Objective:

In this project, you will implement and analyze three types of generative models discussed in class.

Your goal is to understand how each model works, how they are trained, and how the generated outputs vary across different methods. You will explore implementation challenges, hyperparameter tuning, and visualize the outputs.

Task 1: Implement and analyze a basic autoencoder.

  • Implement a basic autoencoder using a simple fully connected network, train it on MNIST. Use different latent dimension (i.e., number of hidden units) sizes (e.g., 2, 8, and 32).
  • Visualize reconstruction results from each version of the autoencoder (e.g., 2, 8, 32 number of hidden units), how does increasing the latent dimensionality affect reconstruction?



As seen in the images, as the latent space dimensions increase, the “bottleneck” gets wider, and the generated numbers become crisper/sharper. If you compare the 2's, 5's, and 8's reconstruction across each dimension, you can clearly see the improvement.

  • Sampling from the learned latent space of the autoencoder. For each version (e.g., 8 and 32), sample random vectors and decode them using the trained decoder. Visualize the sampled outputs. Compare how meaningful the generated digits are at different latent dimensionalities and provide a discussion.

The visualization demonstrates that as the latent dimensionality increases, the model's generative detail improves. At dimension 2, the decoder produces under-determined 'blobs' that represent a mean-average of the dataset. At dimension 8, the decoder still produces “blobs” but with a couple of recognizable digits. By dimension 32, the increased capacity allows the model to capture fine-grained structural details, leading to sharper stroke definitions and somewhat recognizable digit representations.

Task 1: Implement a deep convolutional GAN.

  • Generator. The generator maps the noise variable (e.g., a length-d vector) to a RGB image (e.g., 64x64). The basic block of the generator includes a transposed convolution layer, followed by a batch normalization and a ReLU activation.
  • Discriminator. The discriminator is a convolutional network network similar to what has been implemented in Project 1. One difference is that a leaky ReLU can be used instead of standard ReLU. Leaky ReLU gives a non-zero output for a negative input, aiming to fix the “dying ReLU” problem.
    • The basic block of the discriminator is a conv layer, followed by a batch normalization and a leaky ReLU activation.

The worst part of WGAN is the Gradient Penalty, which requires calculating gradients of gradients and is computationally expensive. In a standard GAN, the Discriminator is updated once and the Generator once. In a WGAN, the Critic was updated 5 times for every 1 Generator update.
